【你的任务】

基于当前工单技能、当前目标、当前步骤位置和已有槽位，先判断当前服务是否仍值得继续规划；若仍可继续，再输出后续宏观计划。

【槽位设计】

- `expected_slots` 里每个槽位必须包含 `name` 和 `description`：
  - `name`：槽位名称，必须严格使用当前绑定工具的参数名或返回字段名，不要自造同义字段
  - `description`：该槽位在业务中代表什么、何时算已确定
- 只设计后续规划、执行和最终结算真正需要的槽位；中间变量、展示字段、内部字段一律不写
- 槽位应覆盖用户目标所需的完整业务信息（如对象、原因、证据等）

【预填槽位】

- 在开始规划步骤之前，先检查用户消息和已有槽位中是否能直接提取到关键业务信息：
  - 用户当前消息中明确给出的信息（如订单号、商品名、问题描述等）
  - 已有 slots 中已经确定的信息
- 将能直接确认的信息立即填入 `slots`，格式为 `{{"槽位名": "槽位值"}}`，key 必须来自 `expected_slots` 的 name
- 只有无法从现有信息中提取的槽位，才需要安排步骤去获取

【步骤描述】

每个步骤只描述：
- `goal`：这一大步要完成什么
- `completion_signal`：这一大步何时算完成
- `target_slots`：这一步需要获取并填充的目标槽位字段名列表，来自 `expected_slots` 中 name 的子集，是本步骤要攻克的槽位
- `available_tools`：当前步骤允许使用的真实业务工具名

【规划规则】

- 只规划宏观主路径，不要把执行时才决定的追问、重试、交互细节拆成多个步骤。
- 能合并的信息收集、查询或确认，应尽量合并成更少的宏观步骤。
- `completion_signal` 不能只写"查询成功"或"拿到字段"，必须写出这一步真正成立的业务条件。
- 技能中规定、但 executor 无法仅靠稳定槽位或工具返回自行推断的限制，必须写进 `goal` 或 `completion_signal`。
- 订单、商品、工单等目标业务对象必须经过用户确认；列表唯一命中或工具查询结果只能作为候选，不能直接算完成。
- `target_slots` 中的每个字段名必须在 `expected_slots` 中存在。
- `target_slots` 只保留后续会复用的稳定字段；字段名必须来自当前绑定工具的参数名或返回字段名，不要发明同义字段、展示字段或底层接口字段。
- 如果当前用户目标与本技能不匹配，`steps` 输出空数组，并在 `reason` 说明原因。
- `当前步骤序号` 就是本轮重新规划起点；只规划从该步骤开始的后续步骤。
- 当前步骤之前的已存在步骤由系统保留，不要重复规划。
- 严格遵守运行时上下文里的 `剩余任务规划步数`。
- 所有推断、结论、决策必须能从现有可用的信息里找到依据，不可以编造，不可以猜测。

【重新规划】

- 若 `上一步执行情况` 中有失败步骤，先分析 `tool_calls_and_results` 中的工具返回结果，判断能否绕开或换工具路径。工具返回已能满足步骤目标时，不再重复调用，直接基于已有结果规划下一步。
- 若仍能形成新的有效路径，则重新规划步骤；若无法绕开，`steps` 输出空并在 `reason` 说明原因。
- **重新规划时 `goal` 格式**：先简短说明此前尝试了什么、卡在哪里，再说现在要做什么。例如："此前通过get_user_orders获取了订单列表并让用户确认，但用户未明确选择具体订单。现在需要引导用户明确目标订单和退货原因。"

【输出要求】

只输出一个符合下列结构的 JSON：

```json
{{
  "reason": "规划依据；如果无法继续，说明失败或取消原因",
  "expected_slots": [
    {{"name": "工具字段名", "description": "该槽位的业务含义和确定条件"}}
  ],
  "slots": {{"槽位名": "槽位值"}},
  "steps": [
    {{
      "goal": "宏观目标",
      "completion_signal": "这一步何时算完成",
      "target_slots": ["来自 expected_slots 中 name 的子集"],
      "available_tools": ["当前步骤可用的真实业务工具名"]
    }}
  ]
}}
```

【硬限制】

- 只输出 JSON。
- 能继续执行时，`steps` 必须非空。
- 无法继续执行时，`steps` 必须为空，且 `reason` 必须直接说明原因。
- `available_tools` 只允许填写当前绑定工具中的名称。
- `target_slots` 中的每个字段名必须在 `expected_slots` 中存在。
